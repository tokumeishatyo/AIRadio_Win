# W13.5 — 曜日替わりメイン DJ + コーナー頭の二層化（挨拶整理・時報リード文）

Mac 版 `s13.5`（`../AIRadio_Mac-main/docs/specs/s13.5-weekday-main-dj.md`）の Windows 移植。
2 つの要望を 1 スライスで実装する（両者は「その日のメイン DJ」を軸に絡むため不可分、Mac 同様）。

1. **曜日替わりメイン DJ**: メインが日替わりで番組全体（OP・ED・時報リード文・トーク）を仕切り、サブが相槌・応答で返す。
2. **コーナー頭の二層化**: 「冒頭コーナーだけ挨拶、他コーナーは時報＋コーナー名」。W13 までは全トークが毎回フル自己紹介
   から始まる（台本プロンプトに番組内位置・時刻が無い）問題の根治。

> 進め方は確立パターン（CLAUDE.md §7）: 本 spec 凍結 → ユーザー承認 → 実装 → 多角レビュー Workflow →
> `dotnet test` 緑 → commit/push → Confluence ミラー。

---

## 0. スコープ

### 入れる（Mac s13.5 §2〜§7）
1. **`WeeklyCast`**（曜日 → 順序付き DJ、先頭＝メイン）。`program.yaml` の `weekly_cast`（省略時は確定表）。
2. **OP / ED の DJ 別固定口上**（`DjSpiel` / `ThemedSegment`。`themes.yaml` の `opening`/`ending` を `by_dj` 構造へ。news は単一のまま）。
3. **コーナー頭の二層化**: 時報リード文（`corners.yaml` の `lead_in`、発話直前に時刻展開・合成）＋ 冒頭トークの挨拶＋出演者紹介（`CornerContext.greeting`）。
4. **主導/応答ダイナミクス**: cast 順序付き（先頭＝メイン）、`DialogueScriptGenerator` に `greeting` 分岐を追加。

### 入れない（後続スライスへ defer）
- **「今日の気分」一言（標準13項目）** → W16 ／ **挨拶での具体時刻断定の抑制** → W16/W17（実機で出たら畳む）。
- **長期記憶 `journalContext`** → W18 ／ **ゲストコーナー** → W14 ／ **アーティスト特集** → W15 ／ **記念日 `DailyCalendar`** → W17（日付季節は W12 `SeasonPhrases` のまま）。
- **ED のショート掛け合い化**（BGM 上の複数話者発話）→ 将来。ED は固定口上をメインが読む。

> Mac の `CornerEngine.swift` / `DialogueScriptGenerator.swift` / `BroadcastEngine.swift` は上記後続機能（guest/artist/journal/今日の気分/深夜抑制）が蓄積済み。W13.5 は **s13.5 §2〜§7 の素のコア**だけを移植する（§3-5）。
>
> **既に Win にあるもの（再実装しない）**: `djs.yaml` の春日部つむぎ（id `tsumugi` / speaker 8）は配置済み（s13.5 §3 は対応不要）。`DialogueScriptGenerator` のメイン主導指示・語尾伝染防止・「番組の途中」制約は W12 で移植済み。`CornerEngine` の `_timeZone`・`TimePhrases`・`TemplateExpander` も既存。

## 1. WeeklyCast（曜日替わり編成、`AIRadio.Core/Program.cs`）

確定表（Mac `WeeklyCast.standard` 一致。先頭＝メイン）:

| 曜日 | メイン | サブ |
|---|---|---|
| 日 | zundamon | metan, tsumugi（**3 人**）|
| 月 | zundamon | metan |
| 火 | metan | tsumugi |
| 水 | tsumugi | zundamon |
| 木 | zundamon | metan |
| 金 | metan | tsumugi |
| 土 | tsumugi | zundamon |

```csharp
public sealed class WeeklyCast
{
    public IReadOnlyDictionary<DayOfWeek, IReadOnlyList<string>> Casts { get; }
    public WeeklyCast(IReadOnlyDictionary<DayOfWeek, IReadOnlyList<string>> casts);

    /// 指定日の編成（先頭＝メイン）。未定義の曜日は空配列。
    public IReadOnlyList<string> DjIds(DateTimeOffset date, TimeZoneInfo? timeZone = null);
    /// 指定日のメイン（編成の先頭）。未定義なら null。
    public string? MainDjId(DateTimeOffset date, TimeZoneInfo? timeZone = null);

    public static WeeklyCast Standard { get; }   // 上表
}
```

- 曜日は `TimeZoneInfo.ConvertTime(date, tz).DayOfWeek` で解釈（既定 `Local`）。乱数なし・完全決定論（テストは固定日付）。
- **Mac は `Calendar.weekday`（1=日…7=土）の Int キー**だが、Win は C# の `DayOfWeek` enum を直接キーにする（意味は同一、+1 変換を避ける）。
  - **実装注意**: `Standard` は **DayOfWeek の名前**で記述する（`Sunday=[zundamon,metan,tsumugi]` …）。Mac の 1 始まり Int をそのまま 0 始まりの `DayOfWeek` キャストに貼ると全曜日が 1 つズレる。
- `weekly_cast` 省略時は `Standard`。ニュースは曜日に関係なく `program.news.dj_id`（龍星）固定（従来どおり。**cast/main を入れても news は `segment.DjId ?? anchor` のまま＝main にしない**）。

## 2. OP / ED の DJ 別固定口上（`AIRadio.Core/Program.cs`）

メインが日替わりになるため、OP/ED の文言・タグラインを **DJ 別に持つ**。BGM 演出（track_uri/intro/volume/ducked/outro）は**共有**。
`{greeting}` 等の時刻プレースホルダは従来どおり発話直前展開。**LLM 生成ではなく固定文言**。

```csharp
/// OP / ED の DJ 別固定口上。
public sealed record DjSpiel(string Announcement, string? Tagline = null);

/// テーマ系セグメント（OP / ED）。BGM 演出は共有、口上は DJ 別。
public sealed record ThemedSegment(ThemeConfig Staging, IReadOnlyDictionary<string, DjSpiel> ByDj)
{
    /// メインを優先、無ければ fallbacks の順、それも無ければ任意の 1 件（Mac `spiel(preferring:fallbacks:)` 一致）。
    public DjSpiel? Spiel(string preferring, IReadOnlyList<string>? fallbacks = null);
}
```

`BroadcastThemes`（`Program.cs`）を変更:
```csharp
// 旧（W7/W8, フラット）: Opening:ThemeConfig + OpeningAnnouncement:string + News + Ending:ThemeConfig + EndingAnnouncement:string + Greetings
// 新（W13.5）:
public sealed record BroadcastThemes(
    ThemedSegment Opening,   // ← ThemeConfig + announcement から ThemedSegment へ
    ThemeConfig News,        // 単一のまま（announcement は実行時注入）
    ThemedSegment Ending,    // ← 同上
    Greetings Greetings);
```

- `Staging.Tagline` は無視（OP は発話直前にメインの tagline を載せる、ED は tagline なし）。`ThemeConfig` は record なので
  `Staging with { Tagline = spiel.Tagline }` で載せる。
- **この型変更は `BroadcastEngine.PerformAsync` の OP/ED を §7 のとおり同時に書き換える**（現在の `themes.Opening`/`themes.OpeningAnnouncement`/`themes.Ending`/`themes.EndingAnnouncement` の直接読みは消える＝3 箇所コンパイルエラー）。フィールド名変更ではなく、**spiel 解決 + `Staging with {Tagline}` + 読み手を `anchor` → `main` へ**の置換である（§7）。

## 3. コーナー頭の二層化（`AIRadio.Core/CornerEngine.cs`, `Dialogue.cs`）

各コーナーを「**頭出しリード文（任意・時刻展開・発話直前合成）**」＋「**本編ダイアログ（LLM・事前合成）**」に分ける。

- **冒頭トーク（番組最初の talk）**: リード文なし。本編ダイアログでメインがまず時刻連動の挨拶（`{greeting}` 由来）→ 番組名 → 出演者紹介 → 本題。
- **その他のトーク・お便り**: 定型リード文「{ampm}{hour}時{minute}分になりました。ここからは〈コーナー名〉です。」を**メインが読む**（ペルソナ語尾なし）。本編は挨拶・自己紹介・番組名なしで即本題。
- **ニュース**: 既存のまま（時報イントロを持つ）。

**時刻の正確さ（最重要・W8/W11 の教訓）**: リード文の時刻は再生時点で正確である必要がある。準備はローリングで数分〜数時間
先行するため、**リード文は LLM 台本に埋め込まず・事前合成もせず**、`CornerEngine.RunAsync`（perform）で**発話直前に
時刻展開→その場で合成**する（news/OP の `Expand()` と同方針）。短文ゆえ合成 1〜2 秒、曲終わりの pause 直後に小さな間が出る程度で許容。

### Core 型変更
- 新規 `CornerContext`（`prepare` に渡す）:
  ```csharp
  public sealed record CornerContext(
      IReadOnlyList<string>? CastDjIds = null,  // 順序付き・空/null なら corner.DjIds
      string? Greeting = null,                  // 冒頭トークのみ非 null
      string? LeadIn = null);                   // その他コーナーの時報リード文テンプレート
  ```
- `CornerTemplate` に `LeadIn`（`string?`、末尾任意）を追加（`corners.yaml` の `lead_in`。空/未指定はリード文なし）。
- `PreparedCorner` に `LeadIn`（`string?`）と `LeadInSpeakerId`（`int`、既定 0）を追加（run で発話直前合成）。
- `CornerEngine.PrepareAsync(corner, djs, CornerContext? context = null, ct = default)`:
  - cast を `context.CastDjIds`（空/null なら `corner.DjIds`）で解決（**順序維持＝メイン先頭**）。
  - **`PreparedCorner.CastDjIds` には解決に使った `castIds`（= `context.CastDjIds` 非空ならそれ、空なら `corner.DjIds`）を格納する**（現状の `CornerEngine.cs` は `corner.DjIds` を入れている＝要変更。run 側の行→話者ルーティングが当日 cast の順序＝メイン先頭に従うため、ここを変えないと s13.5 の主導ダイナミクスが run で崩れる）。
  - `greeting` を `DialogueScriptGenerator` へ渡す。`leadIn`（context.LeadIn 非空）と**メイン speaker**（cast[0].SpeakerId）を `PreparedCorner` に格納。
  - lead_in は **事前合成しない**（テンプレートのまま保持）。W13.5 の lead_in は時刻プレースホルダのみ（`{theme}`/`{guest}` 置換は W14/W15）。
- `CornerEngine`（perform）: `prepared.LeadIn` が非空なら本編の前に **`TimePhrases.Values(_clock.Now, null, _timeZone)` + `TemplateExpander.Expand` で展開 → `_tts.SynthesizeAsync(text, LeadInSpeakerId)` → `_audio.PlayAsync`**。`CornerEvent.LeadIn(text)` を発火。一過性失敗はスキップ（fail-tolerant）、**キャンセルは伝播**（§3-1）。
- `CornerEvent.LeadIn(string Text)` を追加。
- 互換: 単発 `RunAsync(corner, djs, ct)` は `PrepareAsync(corner, djs, ct: ct)`（context 既定）を呼ぶ。

## 4. 主導/応答 + 冒頭挨拶（`AIRadio.Core/DialogueScriptGenerator.cs`）

`GenerateAsync` / `MakeRequest` に `string? greeting = null` を追加（cast は既に順序付きで `djs[0]` をメイン扱い。主導指示は W12 で実装済み）。

- **非 null（冒頭トーク）**: 制約に「これは番組の最初のコーナー。メイン『{main}』がまず『{greeting}』とリスナーに挨拶し、番組名『ケイラボAIラジオ』と本日の出演者（『{names}』）を紹介してから本題に入る。」を追加。**W12 で常時付与している「番組の途中…即本題」制約は greeting 非 null のとき出さない**（分岐に変更）。
- **null（その他）**: 既存の W12 制約を **そのまま（完全な文字列で）else 側へ**移す。Win の現行文言は Mac より長く、「このあとも別のコーナーが続くので、番組全体を締めくくる言い方（『本日最後の曲』『ラストナンバー』『また来週』『お別れの時間』など）はしない。」という**番組終了風抑止句**を含む（W12 の実機修正）。Mac の短い文言に置換せず、この抑止句を維持する。
- 「今日の気分」「深夜抑制」「前回の振り返り（journalContext）」は **W16/W17/W18 へ defer**（本スライスでは付けない）。

## 6. ローダ（Infra）

- `ThemesConfig`: 現行の単一 `MapTheme` を **2 系統に分割**（Mac `staging`/`buildThemed`/`buildSingle` 一致）。
  - 共有 `staging`（`Tagline = null`、`track_uri` 欠落 fail-fast）。
  - `BuildThemed`（opening/ending）: `by_dj`（`Dictionary<string, SpielDto>`）必須 →各 spiel の `announcement` 欠落 fail-fast → `ThemedSegment(staging, byDj)`。**OP/ED の staging は tagline を持たない**（現行 `MapTheme` は全セグメントに `Tagline` を載せているので、OP/ED は載せないよう分岐する）。
  - `BuildSingle`（news）: **tagline 保持**・`by_dj` 不要・announcement は実行時注入のため任意。
  - `BroadcastThemes(Opening, News, Ending, Greetings)` を返す。`greetings` は現行どおり（キー欠落のみ既定）。
- `ProgramConfig`: `weekly_cast`（`Dictionary<string, List<string>>` DTO）を読み、**小文字化した曜日名**（sunday..saturday）→ `DayOfWeek` にマップして `WeeklyCast` を構築 → `ProgramBlueprint.WeeklyCast` に。**省略（null）時は `WeeklyCast.Standard`**、不正曜日名・空編成は fail-fast（Mac `parseWeeklyCast` 一致）。**注意**: `IgnoreUnmatchedProperties` のため DTO プロパティ名の取り違えは黙って null（=Standard）に倒れる→`WeeklyCast`→`weekly_cast` の対応を正例テストで担保。
- `CornersConfig`: `lead_in` を `CornerTemplate.LeadIn` に（**既定 null**。Mac は `?? ""` だが Win は null＝意図的差異。エンジンは「非空のみリード文」判定なので等価）。
- `DjsConfig`: 変更なし。
- `ProgramBlueprint` に `WeeklyCast WeeklyCast`（既定 `Standard`）を追加。`ProgramPlan` は cast 非依存・不変（生成規則は W13 のまま）。

## 7. BroadcastEngine 配線（`AIRadio.Core/BroadcastEngine.cs`）

- 放送開始時（preflight 後）に当日 cast を解決: `cast = plan.Blueprint.WeeklyCast.DjIds(_clock.Now, _timeZone)` を `djs` へ写像（**空・未定義 id は fail-fast**）。`main = cast[0]`。`anchor` は従来どおり（spiel フォールバック・news 既定）。
- **OP / ED**: 読み手＝`main`。spiel＝`themes.Opening/Ending.Spiel(preferring: main.Id, fallbacks: [anchor.Id])`（無ければ空口上）。`staging = Staging with { Tagline = spiel.Tagline }`、`ThemeSequencer.RunAsync(staging, Expand(greetings, spiel.Announcement, extra), main.SpeakerId, ct)`。
- **news**: 従来どおり（`segment.DjId ?? anchor` が読む。曜日の影響を受けない）。
- **talk 準備**: `StartPreparation` の talk で `CornerContext` を作って渡す。
  - 当日 cast の id 列を `CastDjIds` に。
  - **最初の talk セグメント**（`FirstTalkIndex(plan)`、通常 index 2）にのみ `Greeting = TimePhrases.Values(_clock.Now, themes.Greetings, _timeZone)["greeting"]`。
  - それ以外の talk は `LeadIn = corner.LeadIn`。
  - `_cornerRunner.PrepareAsync(corner, djs, context, token)`。
- `FirstTalkIndex(ProgramPlan)`: 先頭から最初の `Talk` セグメントの index（Mac `firstTalkIndex` 移植。安全弁付き）。
- ローリング準備・ED で終了・per-index CTS（W13）は不変。

- **`ThemesConfig`**: opening/ending の by_dj 読み込み（tagline/announcement）、news 単一維持、track_uri 欠落エラー、by_dj 欠落エラー。
- **`ProgramConfig`**: weekly_cast 読み込み（曜日名→DayOfWeek）、省略時 Standard、不正曜日名エラー、空編成エラー。
- **`CornersConfig`**（`DjsCornersConfigTests.cs`）: lead_in あり→`LeadIn` 格納、なし→**null**（既存の「読み飛ばし」アサートを置換）。
- **`DialogueScriptGenerator`**: greeting 非 null で挨拶＋番組名＋出演者紹介、null で挨拶抑制、cast 先頭＝メイン名がプロンプトに入る、主導指示（既存）維持。
- **`CornerEngine`**: context.CastDjIds で cast 上書き（順序）、leadIn を perform で発話直前に時刻展開・合成・再生（FakeClock で時刻確認）、冒頭は leadIn なし、lead_in 一過性失敗で本編継続（fail-tolerant）/ キャンセル伝播。
- **`BroadcastEngine`**: 当日 cast 解決（曜日別）、OP/ED がメインの spiel＋メイン speaker、talk cast＝当日 cast、最初の talk のみ greeting・他は leadIn、未定義 cast id の fail-fast、**ニュースは曜日（メイン）に依存せず `news.dj_id`（龍星）固定**（異なる曜日＝異なるメインでも news 読み手が変わらないことをアサート）。
- **既存保証**（{first_song}・critical OP・キャンセル静寂・ED で終了・ローリング窓上限・news 出現毎）は不変。

## 11. 確定設計判断（W13.5）

1. **`WeeklyCast` は `DayOfWeek` キー**（Mac の Calendar weekday 1..7 Int を C# enum に置換。+1 変換を避け意味同一）。`Standard` を Core 内に持つ（Mac 一致）。
2. **`BroadcastThemes` を ThemedSegment 化**（OP/ED のみ。news は単一）。W7/W8 のフラット形を置換し、OP/ED は main の spiel で読む。
3. **`CornerContext` は Core record（新規型 1 つ）**。`CornerEngine` 具象は維持（§3-5、新 interface 作らない）。`PrepareAsync` に optional 引数で追加し既存呼び出しを最小改修。
4. **lead_in は事前合成せず perform で発話直前展開・合成**（時刻正確性。W8/W11 と同方針）。一過性失敗は fail-tolerant、キャンセルは伝播（§3-1）。
5. **冒頭挨拶は greeting 分岐のみ**（今日の気分=W16 / 深夜抑制=W16,W17 / journal=W18 は前借りしない）。つむぎは djs.yaml 配置済みゆえコード変更なし。

Mac リファレンス: `Program.swift`（WeeklyCast/DjSpiel/ThemedSegment）/ `BroadcastEngine.swift`（cast/main/spiel/cornerContext）/ `CornerEngine.swift`（prepare context・run leadIn）/ `DialogueScriptGenerator.swift`（greeting）/ `ThemeConfigLoader.swift`（by_dj）/ `ProgramConfig.swift`（weekly_cast）。
